# System Design (Sweet Codey)

Curated system design notes and diagrams taken from the Sweet Codey course.

This repository collects concise summaries, design goals, trade-offs, and worked examples for common large-scale system design topics. It's useful as a study guide, reference, or starting point for interview preparation.

## What you'll find here

- High-level concepts and principles for building scalable systems.
- Example designs (YouTube, Instagram feed, tiny URL service, rate limiter, WhatsApp messaging).
- Networking, databases, caching strategies, and communication patterns.

## How to use these notes

- Read top-to-bottom for a gentle progression: fundamentals, networking, storage, then example systems.
- Use the example system documents to practise whiteboard explanations and to create diagrams.
- Add your own notes or diagrams as you learn; files are plain Markdown so they're easy to edit.

## Contributing

Contributions are welcome. A few guidelines:

1. Open a branch for your change and name it descriptively (e.g., `fix/readme-links`, `add/edge-cases-rate-limiter`).
2. Keep changes small and focused. Prefer separate PRs for new example systems or major rewrites.
3. Use clear headings and keep content concise. Add diagrams as images in `public/` and reference them from the markdown.
4. If you update a file name, update the Table of Contents in this `README.md`.

## Notes and style

- Files use simple Markdown. Keep headings, lists, and code blocks readable.
- Prefer illustrative ASCII diagrams or PNG/SVG files in `public/` for complex architecture diagrams.
